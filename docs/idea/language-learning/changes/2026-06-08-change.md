# Change Log — 2026-06-08

**Topic:** Module practice/test loop — coverage guarantee and continuous mastery updates

## Context

Brainstorming surfaced a gap in the original module flow: practice was a single fixed-size session (15 exercises drawn from a ~50-exercise bank), with no guarantee that every vocabulary item in the module was actually shown to the user before the Module Test unlocked. Combined with the test's "50% fresh exercises (not seen during practice)" rule, this meant a user could be tested on vocabulary they had never encountered — an achievement-test validity problem (the test should assess what the module taught, not general proficiency).

The chosen fix addresses the root cause directly: guarantee full vocabulary coverage during practice. Once that holds, "seen during practice" and "every module vocabulary item" become the same set, and the test's exercise-selection no longer needs a special "fresh vs. repeat" rule to stay valid — it can simply draw from the shared bank using the existing mastery-aware algorithm.

## Changes to `idea.md`

### §3.1.1 — Module Execution Flow

- **Mastery updates are now continuous.** Previously mastery scores were updated only during the Module Test, not during practice. Now every exercise — in practice (Step 2) and in the Module Test (Step 3) — updates the mastery score of its linked vocabulary item or grammar concept identically. This also brings the system closer to "true SRS" (relevant to OQ-02).
- **Step 2 (Practice) is no longer a single session.** The user repeats `practiceSessionSize`-sized sessions until **every vocabulary item in the module has appeared in at least one exercise** (shown, not necessarily answered correctly). Grammar concepts are intentionally excluded from this gate — they're already introduced explicitly in Step 1.
- **New convergence guarantee.** Each practice session must dedicate at least `practiceMinUnseenVocabPercent` (default 50%) of its exercises to vocabulary items the user hasn't yet encountered in this module — overriding the mastery-based deprioritization rule for unseen items. This turns "probably converges eventually" into a hard bound (e.g., a 30-item module converges in at most 3 sessions of 20 at 50%).
- **Practice session size increased**: `practiceSessionSize` default 15 → 20.
- **Module Test unlock condition changed.** `testUnlockDelayHours` now counts from the moment Step 2 is *complete* (full vocabulary coverage reached), not from the end of a single practice session.
- **Module Test size increased**: from a fixed 20 questions to **30–40 questions**.
- **Removed the "50% fresh / 50% may repeat" test composition rule.** The test now draws from the same bank as practice using the same mastery-aware selection algorithm (§3.4.3), with no special-cased split. This is now safe because Step 2's coverage guarantee ensures every vocabulary item — fresh or repeated — is necessarily known material.
- **Module Test no longer needs to cover every vocabulary item** — unlike practice, it's explicitly a sample-based check, not an exhaustive re-walk.

### §3.1.2 — Configurable Parameters

- `practiceSessionSize`: default changed 15 → 20; description reworded to reflect that practice may span multiple sessions.
- **Added** `practiceMinUnseenVocabPercent` (default 50): minimum share of each practice session reserved for unseen vocabulary items; the mechanism behind the convergence guarantee.
- `testUnlockDelayHours`: description updated — now measured from Step 2 completion (full coverage), not from finishing one session.
- **Removed** `testFreshExercisePercent` — superseded by the coverage guarantee; no longer needed.

### §3.4.3 — Exercise Generation Strategy

- **Added** a "Coverage override (practice only)" subsection documenting how `practiceMinUnseenVocabPercent` interacts with (and overrides) the mastery-based deprioritization rule during practice sessions, and clarifying that the Module Test and Level Test are unaffected by this override.

### Confirmed unchanged (discussed and intentionally left as-is)

- Module Tests continue to draw from the **same exercise bank** as module practice (no separate module-level test bank).
- Level Tests continue to use their **own dedicated per-level bank** (~60 exercises, cross-module), independent of individual module banks.
- The mastery-aware selection algorithm itself (§3.4.3 steps 1–5) is unchanged for both practice and tests — selection is "as is," only the practice-time coverage override is new.

## Changes to `data-model.md`

### `UserModuleProgress`

Added two fields to support the new coverage gate:

- `vocabularyItemsPracticed` (`string[]`) — accumulates `vocabularyItemId`s the user has encountered at least once during this module's practice, across however many sessions Step 2 takes. Reaching full coverage of `Module.vocabularyItemIds` is what completes Step 2.
- `practiceCompletedAt` (`timestamp | null`) — set the moment full coverage is reached; this is the timestamp `testUnlockDelayHours` counts from (replacing the previous implicit "after finishing the one practice session" trigger).

No other entities required changes — `default-modules.md` has no references to the affected mechanics.
